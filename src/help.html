Create New GUID

Windows Installer requires GUIDs that have no lower case letters. Tools like UUIDGEN and GUIDGEN generate GUIDs with lower case letters. This simple script generates a GUID and ensures that all the letters are in upper case. The result is placed in a text box for copy/paste into Orca or some other Windows Installer related utility.

Create a New Installation Package

This form creates a blank installation database that validates in Orca with no errors. A new installation package can be created by merging the contents of the template files into a single database and setting a few properties on the database and the summary information stream. The text fields allow you to provide the necessary required information. You can enter your own GUIDs or generate new GUIDs for the fields requiring GUIDs.

The Windows Installer SDK samples include several template files that can be used as the basis of an installation. These files are:

schema.msi
A blank database schema containing unpopulated database tables.
Sequence.msi
This database contains the sequence tables InstallUISequence, InstallExecuteSequence, AdminUISequence, AdminExecuteSequence, AdvtUISequence, AdvtExecuteSequence. The tables are prepopulated with the suggested standard sequence as described in the SDK docs.
UISample.msi
This database contains a simple installation user interface as a collection of dialogs.

Scan a Directory for Components

This form scans a directory tree and minimally constructs rows for the File, Directory, Media, Component, Feature, and FeatureComponents tables. It displays the components and their associated files in a tree view after the directories have been scanned.

This form works in two stages. First construct a directory containing your installation as you would like it organized on the distribution media. Then enter the path of that directory into the form and click the scan button. This scans the directory and automatically creates components for the directory contents. All files in a directory that don't require their own component are collected into a component for that directory.

Once the tree view has been populated with components, enter the name of an existing MSI database and click the save button to insert the necessary rows into the File, Directory, Media, Component, Feature and FeatureComponent tables. If you don't have an existing MSI database, use the create package form to create one.

A single feature named "All" is the parent of all the other features created. A feature is created for each component in the installation. The source and target directory hierarchies are identical.

This is a minimal installation that only copies files. It doesn't automatically detect COM components or enter their registration information into the installation. It doesn't control services, the registry, search for files/applications, etc., etc., etc. Use Orca or some other MSI editor to add that information to the installation database after this form generates your basic installation.

User Interface Dialog Tools

This form allows you to preview the user interface dialogs within an installation database. When a database is opened, the list box is populated with the names of the dialoags in the databse. Select a dialog name and then click the Preview button to display the dialog. To end the display of the dialog click the Clear button. Select the close button before opening another database.

You can also clone an existing dialog into the installation database.
